# 音乐英语游戏系统规范 (Music English Game Spec)

## 为什么 (Why)
移动端英语学习打字体验差，用户难以在碎片化时间中坚持。通过“音乐播放 + 闪烁记忆 + 单词点选”的游戏化机制，能够有效提升用户的学习兴趣，并在无需打字的情况下锻炼听力和语感。

## 改变什么 (What Changes)
- **新增**：跨端（兼容PC和移动端）的Web应用。
- **新增**：音乐卡片列表页，展示歌曲及难度。
- **新增**：游戏主页面，包含音乐片段播放、句子闪烁显示（5秒记忆）、打乱单词池点选拼装句子的核心玩法。
- **新增**：管理后台，支持管理员上传/配置音乐资源，并对音乐的每个学习片段设置时间轴、对应的英语句子、中文翻译及干扰词。
- **新增**：游戏难度分级（不同难度下的干扰词数量或闪烁时间不同）。

## 影响 (Impact)
- 受影响的规范：新系统，无历史规范影响。
- 受影响的代码：新建 Vue3 + Vite + TailwindCSS 工程，采用 LocalStorage 模拟数据持久化。

## ADDED Requirements
### 需求：曲库首页
系统应提供一个音乐列表展示页，用户可浏览音乐封面、名称、难度并选择进入游戏。

### 需求：游戏核心玩法
系统应在用户进入游戏后执行以下循环：
1. 加载并播放当前音乐片段。
2. 屏幕中央同步显示该片段的英语句子，5秒后闪烁并消失。
3. 提供一个打乱顺序的单词池（可包含干扰词）。
4. 用户通过点击单词拼凑原句。

#### 场景：拼凑成功
- **当** 用户拼装完成并与原句一致时
- **那么** 系统播放成功特效，自动加载并播放下一句音乐片段。

#### 场景：拼凑失败
- **当** 用户拼装的句子与原句不一致时
- **那么** 系统提示错误，清空当前已拼装区域，并重新播放该音乐片段供用户再次挑战。

### 需求：后台管理系统
系统应提供管理入口：
1. 音乐管理：添加/编辑/删除音乐（标题、封面、音频链接）。
2. 句子管理：为指定的音乐添加句子片段（开始时间、结束时间、原句、中文翻译、干扰词）。

## MODIFIED Requirements
### 需求：完善本地存储（LocalStorage）的数据管理机制
**Reason**: 测试期间经常需要修改默认数据结构或加载新的测试歌曲，但浏览器 LocalStorage 不易于普通用户清理。这导致用户测试时经常加载出旧的或不完整的数据，影响系统验证体验。
**Migration**: 
1. 在管理后台首页（或曲库首页）增加“一键清空所有数据（Clear All Data）”的危险操作入口。
2. 增加“一键重置为默认数据（Reset to Defaults）”的快捷入口，点击后清除现有缓存并从 `defaultData.ts` 重新灌入 100 多条经过校准的内置测试数据，方便开发、测试与体验。

### 需求：引入自动化歌词对齐（Automated Lyrics Alignment）技术
**Reason**: 后台录入歌曲时，手动通过播放器打点（Capture Time）校准每一句歌词的 Start Time 和 End Time 耗时极长，严重影响内容生产效率，人工校准的颗粒度也很难做到完美同步。
**Migration**: 
1. 调研并集成开源的“强制对齐（Forced Alignment）”技术或相关工具（如 WhisperX、Aeneas 等基于深度学习或 DTW 算法的模型）。
2. 在管理后台提供一键“自动生成时间轴”的功能：用户只需提供纯音频和纯文本歌词，系统能自动在后台调用对齐算法，计算并回填每一句歌词精确的 `startTime` 和 `endTime`，甚至生成音素/词级别的时间戳，从而彻底替代手动打点。

### 需求：修复进入页面时的自动播放 Bug
**Reason**: 用户进入页面时偶尔需要点击返回才触发播放，或者直接进入时无播放、无倒计时，存在初始化或加载时机的 Bug，严重影响游戏体验。
**Migration**: 检查并修复游戏页面音频的加载和初始化逻辑，确保用户进入页面或点击开始时能稳定、立刻触发音乐播放和游戏流程倒计时。

### 需求：补充大量测试数据
**Reason**: 当前测试数据不足以支持充分的体验和测试。
**Migration**: 删除旧的测试数据（或者不能用的失效数据），重新收集或生成至少 100 条有效且能够正常播放的音乐及句子测试数据，并确保能够在前端正常加载。

### 需求：端到端（E2E）全流程测试
**Reason**: 现有的单元测试不足以覆盖完整的游戏交互和音乐播放状态验证。
**Migration**: 编写全面的端到端（E2E）测试用例，覆盖从曲库选择、进入游戏、加载音频、检查播放状态、正确/错误拼装到结算的全流程，确保游戏核心流程和音频系统的稳定性。
